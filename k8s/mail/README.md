# GFTB tenant mail CRs

This directory contains tenant-owned `mail.tinyland.dev/v1alpha1` custom
resources for Great Falls Tool Bus.

The substrate grant lives in `tinyland-inc/blahaj`
`deploy/tenants/great-falls-tool-bus/rbac.yaml`; this repo owns the tenant CRs
that consume that grant. The current namespace is `latoolb-us-production`.

These manifests intentionally do not contain:

- DKIM private keys or Kubernetes Secrets
- password material or `passwordSecretRef`
- DNS records
- Mailman/Postorius/HyperKitty resources

`MailAccount` credentials are generated by the account controller on first
reconcile. DKIM key custody and mail transport rendering stay with the
operator/substrate path recorded on TIN-2379.

Useful commands:

```bash
just mail-cr-validate
just mail-cr-render
GFTB_MAIL_KUBECONFIG=/path/to/latoolb-us-production.kubeconfig just mail-cr-server-dry-run
GFTB_MAIL_KUBECONFIG=/path/to/latoolb-us-production.kubeconfig just mail-cr-apply
```
